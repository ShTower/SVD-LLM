## 摘要

大语言模型（LLM）的快速发展受到其庞大模型规模的制约，这使得 LLM 压缩方法成为实际部署中不可或缺的技术。奇异值分解（SVD）为 LLM 压缩提供了一种有前景的解决方案。然而，当前基于 SVD 的 LLM 压缩方法存在两个关键限制：**（1）截断较小的奇异值可能导致更高的压缩损失；（2）SVD 截断后缺乏对压缩权重的更新机制。** 本文提出的 SVD-LLM 是一种基于 SVD 的训练后 LLM 压缩方法，解决了现有方法的局限性。SVD-LLM 引入了一种**截断感知的数据白化技术**（Truncation-Aware Data Whitening），确保奇异值与压缩损失之间存在直接映射关系。此外，SVD-LLM 采用**带顺序低秩近似的参数更新方法**（Sequential Low-rank Approximation），以补偿 SVD 压缩后的精度下降。我们在 10 个数据集、7 个模型（来自 3 个不同 LLM 系列、3 种不同规模）上评估了 SVD-LLM。结果表明，SVD-LLM 显著优于现有方法，尤其在**高压缩比**场景下表现突出。

## 1. 引言

大语言模型在自然语言理解和生成等广泛任务中展现了卓越的能力。然而，LLM 的普及主要受限于其巨大的资源需求，这推动了 LLM 压缩方法的研究。这些方法通常在训练后阶段执行，无需从头重新训练。

基于**量化**（Quantization）、**非结构化剪枝**（Unstructured Pruning）和**结构化剪枝**（Structured Pruning）的训练后压缩方法已被广泛研究。尽管取得了一定成功，这些方法存在特定硬件依赖、推理加速有限等问题。

相比之下，基于**低秩近似**（Low-rank Approximation）的压缩方法（如 SVD）不受这些限制。此外，通过 SVD 压缩的 LLM 在运行时的 KV 缓存也可以被缩减。

然而，现有 SVD 压缩方法（如 FWSVD、ASVD）存在两个根本问题：

1. **SVD 截断与压缩损失之间的错位**：FWSVD 和 ASVD 都未能建立奇异值与模型压缩损失之间的直接关系。结果，截断较小的奇异值反而可能导致更高的压缩损失。
2. **SVD 截断后缺乏模型参数更新**：随着压缩比增加，需要截断的奇异值数量也增加。为补偿精度下降，需要更新压缩模型的剩余参数，但现有方法缺乏此机制。

本文提出的 SVD-LLM 通过两个关键创新解决了上述问题：

- **截断感知数据白化**：有理论证明支持，确保奇异值与压缩损失直接映射，从而识别哪些奇异值应被截断以最小化损失。
- **带顺序低秩近似的参数更新**：顺序微调解耦后的低秩矩阵，实现全局精度恢复。

## 2. 相关工作

### 大语言模型压缩

LLM 通常包含数十亿参数。传统的模型压缩方法需要大量重新训练资源，因此不适用于 LLM。训练后压缩方法可分为四类：

- **非结构化剪枝**：将 LLM 的个别权重设为零，不改变矩阵形状。但非规则稀疏化难以实现理想的加速或内存节省。
- **结构化剪枝**：移除 LLM 的整个通道或结构化组件，硬件实现更友好。代表工作包括 LLM-Pruner、SliceGPT、BlockPruner。
- **量化**：通过降低权重矩阵精度来压缩模型。类似非结构化剪枝，因缺乏硬件支持和高效的低精度计算内核，量化也难以实现理想的推理加速。
- **低秩近似**：SVD 是其中最有代表性的方法。

### SVD 用于语言模型压缩

SVD 是一种广泛使用的低秩近似技术，通过用两个较小的低秩矩阵近似原始矩阵来减小矩阵大小。

- **DRONE**：为 BERT 等小语言模型实现了最优 SVD 压缩，但需要缓存所有输入激活，对 LLM 而言内存消耗过大。
- **FWSVD**：引入 Fisher 信息衡量参数重要性，但需要复杂的梯度计算，计算和内存需求巨大。
- **ASVD**：通过对角矩阵归一化输入通道对权重的影响。然而，FWSVD 和 ASVD 都未能建立奇异值与压缩损失之间的直接关系。

## 3. SVD-LLM 方法

### 3.1 截断感知数据白化

**动机**：由于输入激活的高方差，直接对 LLM 应用普通 SVD 压缩会导致严重的精度下降。现有方法将 LLM 压缩公式化为以下优化目标：

$$O = \min(||WX - W'X||_F)$$

其中 $W$ 是原始权重矩阵，$X$ 是 $W$ 的激活输入，$W'$ 是压缩后的权重矩阵。

以 ASVD 为例，它从 $X$ 中提取一个对角矩阵 $S_0$，其中每个对角线元素是每个通道的绝对均值。然后使用 $S_0$ 归一化 $X$，将 $WX$ 转换为 $(WS_0)(S_0^{-1}X)$，然后在 $WS_0$ 上执行 SVD。

虽然归一化激活能改善性能，但 ASVD **没有建立奇异值与压缩损失之间的直接关系**。截断最小奇异值并不保证获得最小损失。

**核心设计**：SVD-LLM 的核心思想是引入一种截断感知的数据白化技术，确保奇异值与压缩损失之间存在直接映射。为此，SVD-LLM 强制白化后的激活 $S^{-1}X$ 是标准正交的，即每个通道相互独立：

$$(S^{-1}X)(S^{-1}X)^T = S^{-1}XX^T(S^{-1})^T = I$$

其中 $S$ 通过 **Cholesky 分解** 得到。然后在 $WS$ 上执行 SVD，得到 $U, \Sigma, V$。最后，截断 $\Sigma$ 中最小的奇异值，得到两个低秩矩阵：

$$W'_u = U \times [\text{Trunc.}(\Sigma)]^{\frac{1}{2}}, \quad W'_v = [\text{Trunc.}(\Sigma)]^{\frac{1}{2}} \times V^T \times S^{-1}$$

压缩后的权重矩阵为 $W' = W'_u \times W'_v$。

**理论证明**：

- **引理 3.1**：矩阵 $A$ 的 Frobenius 范数等于其 Gram 矩阵迹的平方根。
- **定理 3.2**（单奇异值截断）：若 $S$ 是 $XX^T$ 的 Cholesky 分解，截断第 $i$ 个奇异值的压缩损失 $L_i = \sigma_i$（即奇异值本身）。
- **推论 3.3**（多奇异值截断）：截断最小的奇异值可获得最低的压缩损失，损失 $L^2 = \sum(\sigma_i)^2$。

与 DRONE 相比，SVD-LLM 实现了相同的理论最优压缩损失，但通过增量更新 $XX^T$ 矩阵（而非缓存全部激活），大幅降低了内存消耗。

### 3.2 带顺序低秩近似的参数更新

**动机**：尽管截断感知数据白化有助于在压缩过程中保持精度，但随着压缩比增加，更多较大奇异值被截断，压缩 LLM 的精度仍可能下降。因此需要更新压缩 LLM 中的剩余参数。

**核心设计**：SVD-LLM 提出了一种 LoRA 微调的变体来恢复精度。

具体来说，对于截断感知数据白化生成的两个低秩矩阵 $W'_u$ 和 $W'_v$，标准 LoRA 会直接对 $W' = W'_u \times W'_v$ 进行微调。但 SVD-LLM 提出在 $W'_u$ 和 $W'_v$ 上分别应用 LoRA：

$$W'_u \leftarrow W'_u + B_uA_u, \quad W'_v \leftarrow W'_v + B_vA_v$$

同时微调 $W'_u$ 和 $W'_v$ 不能保证微调损失下降，因为两者的梯度在微调过程中相互依赖，优化一个矩阵可能干扰另一个矩阵的优化。

因此，SVD-LLM 提出了**顺序低秩近似策略**：

1. **第一轮**：冻结 $W'_v$，用 LoRA 微调 $W'_u$
2. **第二轮**：冻结已更新的 $W'_u$，用 LoRA 微调 $W'_v$
3. **合并**：将 $B_u \times A_u$ 和 $B_v \times A_v$ 分别合并回 $W'_u$ 和 $W'_v$

## 4. 实验与分析

### 基线方法

SVD-LLM 与三组基线进行比较：
1. **SVD 方法**：Vanilla SVD、FWSVD、ASVD、FLAP
2. **其他压缩方法**：LLM-Pruner、SliceGPT、BlockPruner（剪枝）；PB-LLM、BiLLM、OneBit（量化）
3. **从头训练的小模型**：StableLM-3B

### 模型与数据集

- **模型**：LLaMA-7B/13B/30B、LLaMA2-7B、OPT-6.7B、Vicuna-7B、Mistral-7B
- **数据集**：WikiText-2、C4（语言建模）；OpenbookQA、WinoGrande、HellaSwag、Arc_e、PIQA、MathQA（分类）；TruthfulQA、GSM8K（生成）

### 主要结果

1. **不同压缩比下的性能**：SVD-LLM 在所有压缩比（20%~80%）下一致优于 Vanilla SVD、FWSVD 和 ASVD。40% 压缩比时，在两个语言建模数据集上困惑度降低超过 99%，六个分类数据集平均准确率提升超过 400%。
2. **不同 LLM 上的性能**：在 OPT-6.7B、LLaMA2-7B、Mistral-7B、Vicuna-7B 上均表现最优。
3. **更大规模 LLM**：在 LLaMA-13B 和 LLaMA-30B 上同样表现优异。
4. **推理效率**：在 GPU 和 CPU 上均实现推理加速和内存缩减。运行时 KV 缓存内存也得到减少。

### 消融实验

- 截断感知数据白化（W）和顺序低秩近似参数更新（U）各自都有贡献，两者结合效果最佳。

---

## 5. 结论

SVD-LLM 是一种基于 SVD 的训练后 LLM 压缩方法，通过截断感知数据白化和带顺序低秩近似的参数更新两个关键技术，解决了现有 SVD 压缩方法的根本局限。在 10 个数据集、7 个模型上的实验证明，SVD-LLM 显著优于现有 SVD 方法（FWSVD、ASVD）和其他类型压缩方法（LLM-Pruner、SliceGPT、BlockPruner、PB-LLM、BiLLM、OneBit），尤其在高压缩比下优势更加明显。

## 附录关键内容

### A.1 SVD-LLM 伪代码

```
1. 输入：权重 W，激活 X，压缩比 ratio
2. XX^T 矩阵 = sum(x * x^T) for x in X 的每个样本
3. S = Cholesky(XX^T)              # Cholesky 分解
4. W_scaled = W * S                 # 白化后的权重
5. U, Σ, V = SVD(W_scaled)         # 奇异值分解
6. k = int(m * n * ratio / (m + n)) # 计算截断后保留的秩
7. Truncate Σ 保留最大的 k 个奇异值
8. W'_u = U_k * sqrt(Σ_k)
9. W'_v = sqrt(Σ_k) * V_k^T * S^{-1}
10. 顺序 LoRA 微调 W'_u 和 W'_v
11. 返回 W'_u, W'_v
```

### A.4 奇异值谱分析

SVD-LLM 的白化技术使奇异值呈长尾分布 — 大部分奇异值接近零，只有少数较大。这确保截断最小奇异值时损失最小。

### A.5 与 DRONE 的对比

SVD-LLM 与 DRONE 实现相同的理论最优压缩损失，但具有以下优势：
- **内存效率**：增量更新 $XX^T$ 而非缓存全部激活
- **压缩速度**：$O(r^3)$ 对比 DRONE 的 $O(r^2 \cdot n)$
- **数值稳定性**：条件数从 $\kappa(X^T)$ 降低到 $\kappa(XX^T)$

### A.9 压缩 LLM 的生成示例

即使在高压缩比（60%、80%）下，SVD-LLM 仍能生成连贯有意义的文本，而 ASVD 在 60% 压缩比时已经完全丧失生成能力。
